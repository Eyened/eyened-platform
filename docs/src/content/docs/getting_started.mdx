The Eyened platform is a modern web platform for visualization and annotation of ophthalmic images with features like:

- Loading of various image formats including DICOM.
- Convenient system for browsing / filtering loaded studies and images.
- Task system for managing grading tasks.
- Drawing tools for 2D image segmentation of enface images and OCT B-scans
- Images and annotations are rendered in the browser, making it very responsive and easy to set up.
- Authentication system secures the viewer, images, thumbnails and annotations.
- Image enhancements such as CLAHE can be applied on the fly.
- Integrated tools for registration of enface images.
- Automated ETDRS grid placement via AI-based bounds detection and landmark detection (fovea, optic disc) upon insertion of CFI images.

## Quick Setup

The easiest way to get started with the Eyened platform is by configuring and running Docker Compose. 
The platform will then be accessible from any machine within your network.

With this setup you will be able to:
- Run the client / viewer, initially on an empty database.
- Import images and associated metadata via the [Import API](/eyened-platform/importing_data) or Python ORM.

**Prerequisite:** Ensure Docker and Docker Compose are installed.

1. Clone the repository:

    ```bash
    git clone https://github.com/Eyened/eyened-platform.git
    ```

2. Navigate to the docker directory:

    ```bash
    cd eyened-platform/docker
    ```

3. You will need to configure a sql database. 
    If you already have one: just set the correct environment variables.
    Alternatively, set the ports and volume mounts in `docker-compose-database.yaml`
    and start a new mysql server with this command:
    ```bash
    docker compose up -f docker-compose-database.yaml -d
    ```

4. Create a `.env` file with the following variables:

    | Variable | Description |
    | --- | --- |
    | PORT | Port the app will be served on |
    | EYENED_API_SECRET_KEY | A random string |
    | EYENED_DATABASE_HOST | Hostname of the database |
    | EYENED_DATABASE_PORT | Port to access the database |
    | EYENED_DATABASE_USER | Username for database access |
    | EYENED_DATABASE_PASSWORD | Password for database access |

5. Configure storage
    The server needs access to a writable directory mounted under /storage/
    Update the docker-compose.yaml to specify the volume mount in the `server` service.
    ```bash
        volumes:
        # Access to the storage directory 
        # The application will use /storage/segmentations.zarr for segmentations
        # And /storage/thumbnails for thumbnails
        - </path/to/storage>:/storage/
    ```

6. Configure the fileserver
    Routing of files is handled by nginx.
    Update nginx.conf by adding locations for every folder where images are stored.
    Each `location` can later be referenced in the database as a `StorageKey`.
    Make sure the alias for each location is mounted in the docker service.

7. Start the platform:

    ```bash
    USERID=$(id -u) GROUPID=$(id -g) docker compose up -d
    ```

8. Initialize the application
    If this is the first time setting up the platform, you will need to initialize the database.
    ```bash
    docker compose exec -it server bash
    ```

    ```bash
    eorm init-database
    ```

    ```bash
    eorm create-user
    ```
    New users can be added through the user interface.


:::note
**For development there is a slightly different setup procedure
:::



## Accessing the Platform

Once the platform is running the viewer will be available at `http://<HOSTNAME>:<PORT>` from within your network

To manage the platform:

- To view logs: `docker compose logs -f`
- To stop the platform: `docker compose down`
- To start the platform: `USERID=$(id -u) GROUPID=$(id -g) docker compose up -d`

:::caution
Make sure to set up regular back ups for both the MySQL database and storage folders.
:::

:::tip
**Using the eyened_orm Python package**

Install the ORM for database access and batch imports:

```bash
cd orm
pip install -e .
```

Create an environment file with database credentials. The ORM expects variables with the `EYENED_DATABASE_` prefix (e.g. `EYENED_DATABASE_USER`, `EYENED_DATABASE_PASSWORD`, `EYENED_DATABASE_HOST`, etc.). Load it and connect:

```python
from dotenv import load_dotenv
from eyened_orm import Database

load_dotenv("path/to/your.env")  # or set vars in the environment
database = Database()
session = database.create_session()
```

See [ORM Configuration](/eyened-platform/orm/configuration) for full options.

For batch imports, patching, and audit/undo workflows, see [ORM Importer](/eyened-platform/orm/importer).
:::
